The component has two separate i18n concerns:

- **`locale`** — controls date formatting (month names, weekday headers, day `aria-label`s) via the browser's `Intl` API.
- **`labels`** — controls UI strings (button labels, panel titles, range context suffixes).

## The `locale` prop

Accepts any [BCP 47 language tag](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Intl#locales_argument). Defaults to `navigator.language`.

<DocCodeBlock lang="tsx" code={codeLocale} />

## Runtime locale switching

Pass a state variable to switch locale at runtime:

<DocCodeBlock lang="tsx" code={codeRuntimeLocale} />

## The `labels` prop

UI strings are controlled separately via `labels`. The default language is English. Pass `Partial<Labels>` to override any string — the rest fall back to English defaults.

<DocCodeBlock lang="tsx" code={codeLabels} />

You only need to supply the strings you want to override. For example, to translate only the range context suffixes for French:

<DocCodeBlock lang="tsx" code={codePartialLabels} />

For the full `Labels` interface, see the [Types reference](/reference/types/).
